================================================================================
1. INTRODUCCIÓN
================================================================================

Se han implementado mejoras de accesibilidad web siguiendo las pautas WCAG 
(Web Content Accessibility Guidelines) en sus tres niveles de conformidad: 
A, AA y AAA. El objetivo es hacer la web accesible para el mayor número de 
usuarios posible, incluyendo personas con discapacidades visuales, motoras, 
auditivas y cognitivas.

Los cambios se han realizado en dos archivos principales:
- index.html: Estructura semántica y elementos HTML accesibles
- style.css: Estilos visuales con contraste adecuado y usabilidad mejorada

================================================================================
2. MEJORAS NIVEL A - CONFORMIDAD BÁSICA
================================================================================

El Nivel A es el nivel mínimo de conformidad WCAG. Estas son mejoras críticas 
sin las cuales algunos usuarios no podrían acceder al contenido.

2.1 SKIP LINK (Criterio 2.4.1 - Bypass Blocks)
-----------------------------------------------
PROBLEMA: Los usuarios de teclado y lectores de pantalla debían pasar por toda 
la navegación en cada página.

SOLUCIÓN IMPLEMENTADA:
- Añadido enlace "Saltar al contenido principal" al inicio del body
- Clase CSS .skip-link que oculta el enlace hasta que reciba foco
- Al recibir foco (Tab), el enlace aparece en la esquina superior

CÓDIGO HTML:
<a href="#main-content" class="skip-link">Saltar al contenido principal</a>

CÓDIGO CSS:
.skip-link {
  position: absolute;
  top: -100px;
  left: 0;
  background: var(--color-text-primary);
  color: var(--color-text-white);
  padding: var(--spacing-sm) var(--spacing-md);
  text-decoration: none;
  font-weight: var(--font-weight-bold);
  z-index: 10000;
}

.skip-link:focus {
  top: 0;
  outline: 3px solid var(--color-accent);
  outline-offset: 2px;
}

2.2 LANDMARKS ARIA (Criterio 1.3.1 - Info and Relationships)
-------------------------------------------------------------
PROBLEMA: Los lectores de pantalla no podían identificar las regiones 
principales de la página.

SOLUCIÓN IMPLEMENTADA:
- role="banner" en el header
- role="main" y id="main-content" en el main
- role="contentinfo" en el footer
- role="navigation" en las navegaciones principales
- aria-label descriptivos para cada navegación

EJEMPLOS:
<header role="banner">
<main id="main-content" role="main">
<footer role="contentinfo">
<nav aria-label="Navegación principal" role="navigation">
<nav aria-label="Selección de idioma">

2.3 TEXTOS ALTERNATIVOS MEJORADOS (Criterio 1.1.1 - Non-text Content)
----------------------------------------------------------------------
PROBLEMA: Los iconos de Font Awesome/Bootstrap Icons no tenían alternativa 
textual.

SOLUCIÓN IMPLEMENTADA:
- aria-hidden="true" en los iconos decorativos
- <span class="visually-hidden"> con texto descriptivo
- aria-label en los enlaces que contienen solo iconos

EJEMPLO:
<a href="/wishlist" aria-label="Lista de Deseos (0 artículos)">
    <i class="bi bi-heart" aria-hidden="true"></i>
    <span class="visually-hidden">Lista de Deseos</span>
</a>

2.4 CLASE VISUALLY-HIDDEN (Best Practice)
------------------------------------------
PROBLEMA: Algunos textos debían ocultarse visualmente pero estar disponibles  para lectores de pantalla.

SOLUCIÓN IMPLEMENTADA:
Clase CSS que oculta contenido visualmente pero lo mantiene accesible:

.visually-hidden {
  position: absolute;
  width: 1px;
  height: 1px;
  margin: -1px;
  padding: 0;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

USADO EN:
- Labels de formularios (<label class="visually-hidden">)
- Encabezados de sección (<h2 class="visually-hidden">)
- Texto descriptivo en iconos

2.5 ETIQUETAS DE FORMULARIO (Criterio 3.3.2 - Labels or Instructions)
----------------------------------------------------------------------
PROBLEMA: Los campos de formulario carecían de etiquetas visibles o accesibles.

SOLUCIÓN IMPLEMENTADA:
- Todos los inputs tienen <label> asociados mediante el atributo 'for'
- Labels ocultos visualmente con clase .visually-hidden
- aria-label adicional para mayor claridad
- Atributo required y aria-required="true" para campos obligatorios

EJEMPLOS:
<label for="search" class="visually-hidden">Buscar productos</label>
<input type="search" id="search" placeholder="Search..." 
       aria-label="Buscar productos">

<label for="newsletter" class="visually-hidden">Tu dirección de email</label>
<input type="email" id="newsletter" placeholder="Your email address" 
       aria-label="Ingrese su dirección de correo electrónico" 
       required aria-required="true">

2.6 ENLACES DESCRIPTIVOS (Criterio 2.4.4 - Link Purpose)
---------------------------------------------------------
PROBLEMA: Enlaces vacíos (href="") o con texto genérico (#).

SOLUCIÓN IMPLEMENTADA:
- Todos los href="" reemplazados con destinos válidos
- Enlaces "#" reemplazados con URLs reales
- aria-label añadido a enlaces que necesitan contexto adicional

EJEMPLOS CORREGIDOS:
Antes: <a href="">More details</a>
Después: <a href="#promotions" aria-label="Ver más detalles sobre la promoción">More details</a>

Antes: <a href="#">Home</a>
Después: <a href="/">Home</a>

2.7 ESTRUCTURA SEMÁNTICA (Criterio 1.3.1)
------------------------------------------
PROBLEMA: Falta de identificadores únicos para secciones.

SOLUCIÓN IMPLEMENTADA:
- aria-labelledby en todas las secciones principales
- IDs únicos para todos los encabezados de sección
- Estructura jerárquica de encabezados mantenida (h1 > h2 > h3)

EJEMPLOS:
<section class="products" aria-labelledby="products-heading">
    <h2 id="products-heading">Welcome to Store</h2>
    ...
</section>

<section class="blog" aria-labelledby="blog-heading">
    <h2 id="blog-heading">Latest Blog</h2>
    ...
</section>

2.8 NAVEGACIÓN CON MENÚS DESPLEGABLES (Criterio 4.1.2)
-------------------------------------------------------
PROBLEMA: Los menús desplegables no tenían indicadores de estado.

SOLUCIÓN IMPLEMENTADA:
- aria-haspopup="true" en botones que abren menús
- aria-expanded="false" para indicar estado (debería cambiar con JavaScript)
- aria-label descriptivo en cada navegación

EJEMPLO:
<nav class="header-nav-language" aria-label="Selección de idioma">
    <a href="#" class="header-nav-toggle" 
       aria-haspopup="true" aria-expanded="false">English</a>
    <ul>...</ul>
</nav>

2.9 SVG ACCESIBLE (Criterio 1.1.1)
-----------------------------------
PROBLEMA: El SVG de la firma carecía de texto alternativo.

SOLUCIÓN IMPLEMENTADA:
- role="img" en el elemento SVG
- <title> con ID único
- aria-labelledby apuntando al title

EJEMPLO:
<svg width="400" height="180" xmlns="http://www.w3.org/2000/svg" 
     role="img" aria-labelledby="signature-title">
    <title id="signature-title">Firma IPG - Ian Pérez González</title>
    <rect x="50" y="20" rx="20" ry="20" width="150" height="150" 
          class="signature-rect" />
    <text x="125" y="95" class="signature-text">IPG</text>
</svg>

3. MEJORAS NIVEL AA - CONFORMIDAD MEJORADA
================================================================================

El Nivel AA incluye requisitos más estrictos que mejoran significativamente 
la experiencia de usuario. Es el nivel recomendado por WCAG y requerido por 
muchas legislaciones.

3.1 CONTRASTE DE COLOR (Criterio 1.4.3 - Contrast Minimum)
-----------------------------------------------------------
PROBLEMA: Varios elementos no cumplían con el ratio de contraste 4.5:1 para 
texto normal o 3:1 para texto grande.

SOLUCIÓN IMPLEMENTADA:
Ajuste de colores en variables CSS del :root:

ANTES:
--color-text-primary: #333;      (Ratio: ~12.6:1) ✓
--color-text-secondary: #666;    (Ratio: ~5.7:1)  ✓
--color-text-light: #999;        (Ratio: ~2.8:1)  ✗

DESPUÉS:
--color-text-primary: #1a1a1a;   (Ratio: ~16.1:1) ✓✓
--color-text-secondary: #4d4d4d; (Ratio: ~9.0:1)  ✓✓
--color-text-light: #767676;     (Ratio: ~4.54:1) ✓ (AA cumplido)

COLORES PRINCIPALES AJUSTADOS:
--color-primary: #a67c52 (antes #c89d66)
--color-secondary: #6b0f28 (antes #8b1538)
--color-accent: #b8941f (antes #d4af37)

BACKGROUND DEL HEADER:
Antes: background-color: #2a2a2a;
Después: background-color: #1a1a1a; (Mayor contraste con texto blanco)

3.2 INDICADORES DE FOCO VISIBLES (Criterio 2.4.7 - Focus Visible)
------------------------------------------------------------------
PROBLEMA: El foco del teclado era apenas visible o inexistente.

SOLUCIÓN IMPLEMENTADA:
Estilos de foco destacados para todos los elementos interactivos:

*:focus {
  outline: 3px solid var(--color-accent);
  outline-offset: 2px;
}

*:focus-visible {
  outline: 3px solid var(--color-accent);
  outline-offset: 2px;
}

a:focus, button:focus, input:focus, select:focus, textarea:focus {
  outline: 3px solid var(--color-accent);
  outline-offset: 2px;
}

CARACTERÍSTICAS:
- Grosor de 3px para alta visibilidad
- Color dorado (#b8941f) que contrasta con todos los fondos
- offset de 2px para separación del elemento
- Aplicado consistentemente en toda la página

3.3 TAMAÑOS DE FUENTE ESCALABLES (Criterio 1.4.4 - Resize Text)
----------------------------------------------------------------
PROBLEMA: Algunos tamaños de fuente estaban en píxeles fijos.

SOLUCIÓN IMPLEMENTADA:
- Todas las fuentes en unidades relativas (rem)
- font-size-xs aumentado de 0.75rem a 0.875rem
- font-size-sm aumentado de 0.875rem a 1rem
- Base de 16px permite zoom al 200% sin pérdida de funcionalidad

VARIABLES MEJORADAS:
--font-size-xs: 0.875rem;  (14px - Antes 12px)
--font-size-sm: 1rem;      (16px - Antes 14px)
--font-size-base: 1rem;    (16px)

3.4 ESPACIADO DE TEXTO (Criterio 1.4.12 - Text Spacing)
--------------------------------------------------------
PROBLEMA: Espaciados insuficientes para usuarios con dislexia o baja visión.

SOLUCIÓN IMPLEMENTADA:
- line-height: 1.6 en body y párrafos (mínimo WCAG: 1.5)
- line-height: 1.4 en encabezados
- margin-bottom consistente en párrafos y encabezados

CÓDIGO:
body {
  line-height: 1.6; /* WCAG AA: 1.5 mínimo */
}

p {
  line-height: 1.6;
  margin-bottom: var(--spacing-sm);
}

h1, h2, h3, h4, h5, h6 {
  line-height: 1.4;
  margin-bottom: var(--spacing-sm);
}

3.5 MÚLTIPLES FORMAS DE NAVEGACIÓN (Criterio 2.4.5)
----------------------------------------------------
PROBLEMA: Solo existía la navegación principal.

SOLUCIÓN IMPLEMENTADA:
- Navegación principal en header
- Navegación de categorías en sección de productos
- Navegación en footer (About, Shop, Customer, Offers)
- Skip link para acceso directo al contenido
- Todas las navegaciones con aria-label únicos

EJEMPLOS:
<nav aria-label="Navegación principal">
<nav aria-label="Filtros de productos">
<nav aria-label="Enlaces sobre nosotros">
<nav aria-label="Enlaces de tienda">

3.6 ENCABEZADOS Y ETIQUETAS DESCRIPTIVOS (Criterio 2.4.6)
----------------------------------------------------------
PROBLEMA: Algunos encabezados eran demasiado genéricos.

SOLUCIÓN IMPLEMENTADA:
- Todos los h2 tienen IDs únicos
- aria-labelledby conecta secciones con sus encabezados
- Encabezados de navegación del footer mejorados con IDs

EJEMPLOS:
<h2 id="products-heading">Welcome to Store</h2>
<h2 id="blog-heading">Latest Blog</h2>
<h2 id="newsletter-heading">Sign up for newsletters & get all updates</h2>
<h2 id="footer-about">Katora.</h2>

3.7 ELEMENTOS DE FORMULARIO MEJORADOS (Criterio 3.3.3)
-------------------------------------------------------
PROBLEMA: Faltaba información sobre campos requeridos y validación.

SOLUCIÓN IMPLEMENTADA:
- Atributo required en campos obligatorios
- aria-required="true" para lectores de pantalla
- Labels descriptivos (visibles o aria-label)
- Placeholder como ayuda adicional (no como reemplazo de label)

EJEMPLO:
<input type="email" id="newsletter" placeholder="Your email address" 
       aria-label="Ingrese su dirección de correo electrónico" 
       required aria-required="true">

3.8 MEJORAS EN SELECT (Criterio 1.3.5)
---------------------------------------
PROBLEMA: El select de categorías carecía de label.

SOLUCIÓN IMPLEMENTADA:
- Label con clase visually-hidden
- ID único para asociación
- aria-label como respaldo
- Estilos de foco mejorados

CÓDIGO:
<label for="category-select" class="visually-hidden">Categoría</label>
<select id="category-select" class="category-select" 
        aria-label="Seleccionar categoría de producto">
    <option value="">All Categories</option>
    ...
</select>

4. MEJORAS NIVEL AAA - CONFORMIDAD MÁXIMA
================================================================================

El Nivel AAA es el más alto de conformidad WCAG. No es obligatorio para todas 
las páginas, pero mejora significativamente la accesibilidad.

4.1 CONTRASTE SUPERIOR (Criterio 1.4.6 - Contrast Enhanced)
------------------------------------------------------------
PROBLEMA: Se requiere ratio de 7:1 para texto normal y 4.5:1 para texto grande.

SOLUCIÓN IMPLEMENTADA:

TEXTO SOBRE FONDO CLARO:
- Texto principal #1a1a1a sobre #fff: Ratio 16.1:1 ✓ (Supera 7:1)
- Texto secundario #4d4d4d sobre #fff: Ratio 9.0:1 ✓ (Supera 7:1)
- Texto light #767676 sobre #fff: Ratio 4.54:1 (Solo para texto grande)

TEXTO SOBRE IMÁGENES:
- Overlay oscurecido de rgba(0,0,0,0.5) a rgba(0,0,0,0.7)
- text-shadow añadido: 2px 2px 4px rgba(0,0,0,0.8)
- Combinación garantiza ratio superior a 7:1

CÓDIGO:
section.hero {
  background: linear-gradient(to right, rgba(0,0,0,0.7), rgba(0,0,0,0.7)),
    url("../img/Slider_01.webp") center/cover no-repeat;
  
  h2 {
    text-shadow: 2px 2px 4px rgba(0,0,0,0.8);
  }
}

section.perfect-match {
  background: linear-gradient(to right, rgba(0,0,0,0.7), rgba(0,0,0,0.7)),
    url("../img/background_image.webp") center/cover no-repeat;
  
  h2 {
    text-shadow: 2px 2px 4px rgba(0,0,0,0.8);
  }
}

BADGES DE PRODUCTOS:
.badge-new {
  background-color: #0066cc; /* Aumentado de #4a90e2 para mejor contraste */
}

TIMER COUNTDOWN:
span {
  background-color: #404040; /* Aumentado de #6b6b6b */
  border: 2px solid #333; /* Añadido para definición */
}

DEAL SECTION:
span:first-child {
  color: #c1121f; /* Mejorado de #dc3545 */
}

4.2 TAMAÑOS MÍNIMOS DE ELEMENTOS INTERACTIVOS (Criterio 2.5.5)
---------------------------------------------------------------
PROBLEMA: Algunos elementos eran difíciles de clickear/tocar (especialmente 
en móviles).

SOLUCIÓN IMPLEMENTADA:
Variable CSS para tamaño mínimo de clic:

--min-touch-target: 44px;

APLICADO A:
- Todos los botones (min-height: 44px, min-width: 44px)
- Todos los enlaces (min-height: 44px)
- Inputs de formulario (min-height: 44px)
- Elementos de navegación (padding incrementado)

CÓDIGO:
button {
  min-height: var(--min-touch-target);
  min-width: var(--min-touch-target);
  padding: var(--spacing-xs) var(--spacing-sm);
}

a {
  min-height: var(--min-touch-target);
  display: inline-flex;
  align-items: center;
  padding: var(--spacing-xs) 0;
}

input {
  min-height: var(--min-touch-target);
  padding: var(--spacing-sm);
}

ELEMENTOS ESPECÍFICOS:
Hero section botón:
  padding: var(--spacing-md) var(--spacing-xl);
  min-height: var(--min-touch-target);

Navigation toggles:
  min-height: var(--min-touch-target);
  min-width: 80px;

4.3 ESPACIADO DE TEXTO MEJORADO (Criterio 1.4.8)
-------------------------------------------------
PROBLEMA: El espaciado insuficiente dificulta la lectura.

SOLUCIÓN IMPLEMENTADA:

LINE-HEIGHT:
- Párrafos: 1.6 (WCAG AAA requiere mínimo 1.5)
- Encabezados: 1.3-1.4
- Hero h2: 1.3 con text-shadow para legibilidad

LETTER-SPACING:
- Badges: letter-spacing: 0.5px
- Texto en mayúsculas: letter-spacing: 1px

MARGINS Y PADDING:
- Uso consistente de variables de spacing
- margin-bottom en párrafos y encabezados
- Padding aumentado en elementos interactivos

CÓDIGO:
p {
  line-height: 1.6;
  margin-bottom: var(--spacing-sm);
}

h1, h2, h3, h4, h5, h6 {
  line-height: 1.4;
  margin-bottom: var(--spacing-sm);
}

.product-badges span {
  letter-spacing: 0.5px;
}

4.4 IDENTIFICACIÓN DE ENLACES (Criterio 2.4.4)
-----------------------------------------------
PROBLEMA: Los enlaces no siempre eran distinguibles del texto normal.

SOLUCIÓN IMPLEMENTADA:
- Enlaces en contenido principal tienen subrayado por defecto
- text-decoration-thickness para grosor del subrayado
- text-underline-offset para separación
- Grosor aumenta en hover

CÓDIGO:
main a:not(.read-more):not([class*="button"]):not([class*="btn"]) {
  text-decoration: underline;
  text-decoration-thickness: 1px;
  text-underline-offset: 2px;
}

main a:hover {
  text-decoration-thickness: 2px;
}

4.5 EVITAR IMÁGENES DE TEXTO (Criterio 1.4.9)
----------------------------------------------
PROBLEMA: Texto en imágenes no es escalable ni accesible.

SOLUCIÓN IMPLEMENTADA:
- Todo el texto importante está en HTML, no en imágenes
- El logo "Katora." es texto HTML, no imagen
- SVG firma usa elemento <text> (puede escalarse)
- Badges son elementos HTML con clases CSS

4.6 MEJORAS EN BOTONES Y CTAs (Criterio 2.5.5)
-----------------------------------------------
PROBLEMA: Botones pequeños y difíciles de identificar.

SOLUCIÓN IMPLEMENTADA:
- Tamaño mínimo 44x44px aplicado
- Padding aumentado para mejor área de clic
- Border-radius añadido para mejor definición visual
- Box-shadow para profundidad
- Estados hover y focus claramente distinguibles

CÓDIGO EJEMPLO (Hero CTA):
a {
  background-color: var(--color-primary);
  color: var(--color-text-white);
  padding: var(--spacing-md) var(--spacing-xl);
  font-weight: var(--font-weight-bold);
  min-height: var(--min-touch-target);
  font-size: var(--font-size-base);
  text-decoration: none;
  box-shadow: 0 4px 8px rgba(0,0,0,0.3);
  border-radius: var(--border-radius-sm);
}

a:hover {
  transform: translateY(-2px);
  box-shadow: 0 6px 12px rgba(0,0,0,0.4);
}

a:focus {
  outline: 3px solid var(--color-accent);
  outline-offset: 3px;
}

---------------------------------------------------
PROBLEMA: Badges pequeños con bajo contraste.
